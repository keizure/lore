# Transformer Block 的内部机制：残差连接、FFN 与零件全景

上一篇探索了 LLM 从文本到输出的完整分层流程。这篇深入 Transformer Block 内部，回答一个更精细的问题：Block 里那些零件——残差连接、FFN、激活函数——各自解决什么问题，彼此又是什么关系？

---

## 一、残差连接：让深网络「可以被训练」

### 问题根源：梯度消失

网络越深理论上越强，但实践中超过一定深度后，训练反而变得更差。原因不是过拟合，而是**优化本身失败了**：反向传播时，梯度通过链式法则逐层传回，每层乘以一个小于 1 的局部梯度，连乘之后趋近于零。靠近输入的层几乎收不到训练信号，权重无法更新。

```
每层梯度 ≈ 0.9   →   0.9^32 ≈ 0.034   →   接近 0
```

### 解法：直通路保证梯度不消失

残差连接的公式只有一行：`output = F(x) + x`

把子层输入 x 通过一条直通路（shortcut）绕过子层，直接加到输出。反向传播时，该路径对 x 的偏导数恒为 +1，无论网络多深梯度都有一条保底通道。

子层也因此只需学习「调整量」F(x)，而不是从零构建完整映射——学残差比学全量变换容易得多。这一整条持续被各层读写的向量，就是**残差流（Residual Stream）**。

### 残差连接的双重身份

同一个加法结构，在两个阶段扮演不同角色：

- **训练时（反向传播）**：梯度高速公路，让梯度无损传回浅层
- **推理时（正向传播）**：信息叠加总线，每个子层只能往残差流上「写增量」，无法覆盖已有信息

## 二、Block 内为什么有两条残差连接？

一个 Block 内有两条残差连接，分别绕过 Attention 和 FFN。如果残差连接只是为了梯度，一条绕过整个 Block 的就够了——两条的存在有更深的意义。

**Attention 和 FFN 是严格串行的**，不是并列：

```
x → Attention → x' = x + Δ_attn    ← FFN 读取的是这个，不是原始 x
  → FFN       → x'' = x' + Δ_ffn
```

两条独立残差带来三重必要性：

| 必要性 | 说明 |
|--------|------|
| **顺序解耦** | Attention 先聚合上下文，FFN 再在此基础上提取知识，两步有序但各自独立 |
| **信息保护** | 每个子层只能叠加增量，无法覆盖它收到的输入 |
| **梯度独立** | Attention 和 FFN 各有独立的梯度通道，训练信号互不干扰 |

**「看到未被改动的输入」的精确含义**：不是每个子层都能看到原始 x，而是每个子层的残差连接保护「自己这一步的输入」不被自己覆盖。残差连接②让 FFN 无法抹去 x'，不是让 FFN 看到 x。

整个残差流的累积性：

```
x₀ → x₀+Δ_attn₁ → x₀+Δ_attn₁+Δ_ffn₁ → ... → 最终输出
```

每次都是加法，从不是替换。这是残差流最核心的性质：**只增不替**。

## 三、FFN：表达能力 + 知识存储

### 为什么 FFN 有参数，而且是参数最多的部分？

FFN（Feed-Forward Network，前馈网络）是整个 Block 里参数量最大的子层：

```
FFN = 线性层(W1) + 激活函数 + 线性层(W2)
          ↑             ↑              ↑
       有参数         纯函数，无参数    有参数
```

W1、W2（SwiGLU 变体还有 W3）是可训练的权重矩阵，训练时通过梯度下降反复更新。激活函数（ReLU、SiLU）是纯函数，没有任何参数，无论训练多久都不变。

以 Llama 3 8B 为例，FFN 共约 **5.6B 参数**，超过整个模型的一半。

### 非线性从哪里来？

两个线性变换串联等价于一个线性变换，表达能力没有增加。激活函数插在中间打破这一限制：不同输入激活不同的神经元子集，实现真正的非线性——不同输入走不同的「路径」。

### FFN 如何充当知识记忆库？

2021 年论文《Transformer Feed-Forward Layers Are Key-Value Memories》揭示了一个深刻类比：

```
W1 的每一行  →  「键」（key）：匹配某种输入模式
激活函数     →  决定哪些键被命中（点积够高就激活，否则清零）
W2 的每一列  →  「值」（value）：被命中的键输出的内容
```

这和 Attention 的 K/V 是**不同的概念**：Attention 的 K/V 是推理时从当前输入动态计算的激活值；FFN 的「键/值」是训练后固定在权重矩阵里的参数。前者是动态的，后者是静态的。

训练时，大量见到「法国首都是巴黎」后，W1 某行被训练成擅长匹配「法国 + 首都」模式，W2 对应列被训练成偏向「巴黎」语义方向——知识就这样写入参数。推理时参数固定，知识被「冻结」在模型文件里。

### SwiGLU：现代 LLM 的 FFN 变体

主流 LLM（Llama、Mistral、Gemma 等）使用 SwiGLU 替代标准 FFN，将两个矩阵扩展为三个：

```
        x
       / \
     W1   W3
      ↓    ↓
    SiLU (线性)
      ↓    ↓
      └─⊙──┘   ← 逐元素相乘（门控）
          ↓
         W2 → output
```

`SiLU(W1·x)` 是门控，`W3·x` 是线性路径，两者相乘让每个隐藏维度有独立的可学习门，信息控制比 ReLU 的硬 0/1 更精细。SiLU（`f(x) = x·sigmoid(x)`）比 ReLU 平滑，轻微负值时仍有梯度。

## 四、FFN 与 Attention 的分工

FFN 不是 LLM 的「核心」，Attention 也不是——两者缺一不可，分工互补：

```
Attention  →  「算法层」：理解上下文、在 token 间路由信息
FFN        →  「记忆层」：检索存储的知识、做深度语义变换
```

类比：Attention 是图书馆管理员（理解问题、找到相关材料），FFN 是书架（存着所有知识）。单独的书架不会回答问题；单独的管理员手里没有知识。

训练的核心是 **loss**——让预测下一个 token 越来越准确。为此，Attention 和 FFN 协同进化，没有哪个更核心。

## 五、Block 的完整零件图

Block 是 LLM 的基本思考单元，N 个 Block 堆叠构成模型主体：

```
输入残差流 x
     │
     ├──────────────────────────────────────┐
     ▼                                      │ 直通路①
 ① RMSNorm（无参数，稳定数值）              │
     ▼                                      │
 ② Multi-Head Attention                    │
     │   W_Q / W_K / W_V: Linear（有参数）  │
     │   RoPE：注入位置编码（无参数）         │
     │   Softmax(QKᵀ/√d)·V（无参数）        │
     │   W_O: Linear（有参数）              │
     ▼                                      │
 ③ 残差加法  x = x + Attn_out ←────────────┘
     │
     ├──────────────────────────────────────┐
     ▼                                      │ 直通路②
 ④ RMSNorm（无参数）                        │
     ▼                                      │
 ⑤ FFN (SwiGLU)                            │
     │   W1 / W3: Linear（有参数）           │
     │   SiLU 激活（无参数）                 │
     │   W2: Linear（有参数）               │
     ▼                                      │
 ⑥ 残差加法  x = x + FFN_out ←─────────────┘
     ▼
输出残差流 x'  → 下一个 Block
```

| 零件 | 有无参数 | 职责 |
|------|---------|------|
| RMSNorm | 无 | 稳定数值尺度 |
| W_Q / W_K / W_V | 有 | 投影出 Query、Key、Value |
| RoPE | 无 | 注入位置信息 |
| Softmax(QKᵀ)·V | 无 | 注意力加权聚合 |
| W_O | 有 | 多头输出投影回 d_model |
| 残差连接① | 无 | 保护 x，Attention 只写增量 |
| W1 / W3 | 有 | FFN 门控升维 |
| SiLU | 无 | 激活函数，非线性门控 |
| W2 | 有 | FFN 降维，写回知识 |
| 残差连接② | 无 | 保护 x'，FFN 只写增量 |

### Block 在整个模型中的位置

```
Tokenizer → Embedding → [Block × N] → LM Head → 采样
                              ↑
                          思考在这里发生
                          浅层：语法词法
                          中层：句子结构
                          深层：推理事实
```

所有 Block 结构相同，但参数完全独立，训练后自然分化出不同层次的专业化。

## 六、关于卷积层和全连接层

**卷积层**在文本 LLM 里基本消失——Attention 能处理任意距离的 token 关联，卷积的固定局部窗口不够用。多模态模型的视觉编码器部分可能仍用卷积，但那是视觉模块，不是语言主体。

**全连接层**从未消失，只是改了名字。`全连接层 = 线性层 = Linear = W·x + b`，这是同一个东西：

```
Attention 的 W_Q / W_K / W_V / W_O  →  都是全连接层
FFN 的 W1 / W2 / W3                 →  都是全连接层
Embedding / LM Head                 →  都是全连接层
```

Llama 3 8B 有 200+ 个全连接层。「全连接层」这个词消失了，是因为它降格为基础零件融入了 Attention 和 FFN，人们不再单独强调它。

## 附录：探索路径

### Q1 — 残差连接和 FFN 各自解决了什么核心问题？
残差连接解决梯度消失（深网络训练问题），FFN 解决表达能力不足和知识存储问题。公式 `output = F(x) + x` 让梯度有 +1 保底通道，子层只学「调整量」而非完整映射。FFN 通过激活函数提供非线性，通过 W1/W2 权重充当键值记忆库存储事实知识。

### Q2 — Block 内为什么有两处残差连接？
如果只解决梯度消失，一条就够。两条的原因：顺序解耦（Attention 先，FFN 后，各自独立）、信息保护（每层只能叠加增量）、梯度独立（两层的训练信号互不干扰）。残差连接在训练时是梯度高速公路，在推理时是信息叠加总线。

### Q3 — Attention 和 FFN 是并列处理再聚合的吗？
不是，是严格串行的。FFN 的输入是 `x' = x + Δ_attn`，依赖 Attention 的输出。两者都有残差连接导致看起来像并列，但 Δ_ffn 是基于 x+Δ_attn 计算的，不是基于原始 x。

### Q4 — FFN「看到未被改动的输入」是什么意思？
每条残差连接保护的是「自己这步的输入」不被自己覆盖，不是保护整个 Block 的原始输入。「x₀ 的信息被保留」意味着没有任何子层能把 x₀ 从残差流里抹掉（有残差 vs 无残差的对比：有残差时子层只能加，无残差时子层可以完全覆盖）。

### Q5 — x₀ + Δ_attn₁ 合并成一个值，x₀ 就是残差吗？
合并成一个值是正确的。但术语方向反了：F(x) 才是「残差」（子层学习的增量），x 是「恒等映射/shortcut」。F(x) 通常很小，学小的调整比学完整映射容易。x₀ 信息「被保留」指的是没有子层能覆盖它，而非它被单独存储。

### Q6 — FFN 的非线性、知识存储、SwiGLU 与参数
FFN 的参数是 W1/W2/W3（权重矩阵），激活函数无参数。非线性来自激活函数（不同输入激活不同神经元）。知识存储靠 W1/W2 的权重组合（键值记忆系统）。SwiGLU = SiLU(W1·x) ⊙ (W3·x)，门控让每个维度有独立可学习门，比 ReLU 表达能力更强。训练时梯度下降反复更新 W1/W2/W3，推理时固定。

### Q7 — FFN 的键值是 Attention 的 KV 吗？FFN 不是纯函数吗？
FFN 键值是类比，不是 Attention KV。区别：Attention KV 是推理时从当前输入动态计算的激活值；FFN「键值」是固定在权重矩阵里的参数。FFN 不是纯函数（有 W1/W2 参数），激活函数才是纯函数。

### Q8 — 激活函数和 FFN 是一回事吗？
不是。激活函数是 FFN 内部的一个零件：FFN = 线性层(W1) + 激活函数 + 线性层(W2)。激活函数提供非线性，但存储知识的是 W1/W2 参数。没有激活函数，W1/W2 只能表达线性关系。

### Q9 — 大模型的主体和训练核心是 FFN 吗？
FFN 参数量最大（>50%），但 Attention 也有大量参数（~25%）且同样被训练。两者分工互补：Attention 是算法层（理解上下文、路由信息），FFN 是记忆层（检索和加工知识）。训练核心是 loss，Attention 和 FFN 协同进化。

### Q10 — 现代大模型还有卷积层和全连接层吗？
卷积层在文本 LLM 里基本消失（Attention 覆盖任意距离，卷积局部窗口不够）。全连接层大量存在（是整个模型的骨架），只是叫名字变成了 Linear/Projection，融入 Attention 和 FFN 不再单独强调。

### Q11 — Block 的完整结构及其与其他组件的关系
Block = RMSNorm → Attention（W_Q/K/V/O）→ 残差① → RMSNorm → FFN（W1/2/3 + SiLU）→ 残差②。所有 Block 结构相同但参数独立，训练后浅层学语法、深层学推理。整体结构：Embedding → Block×N → LM Head → 采样（自回归循环）。
