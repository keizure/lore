# Lore Skill 设计文档

> 创建日期：2026-05-06
> 状态：待实现

---

## 概述

`lore` 是一个项目级 Claude Code slash command，用于将博客文章或网页内容转化为结构化的中文阅读笔记。它解决了「每次阅读外文技术文章都要重复执行摘要、翻译、概念下钻」的问题，通过渐进式工作流在每个阶段产出可读文件。

---

## 触发方式

```
/lore <URL 或本地文件路径>
```

示例：
```
/lore https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
/lore ./raw/async-article.md
```

---

## 文件位置

- **Skill 文件**：`.claude/commands/lore.md`（项目级，随仓库一起维护）
- **输出文件**：`reading/<slug>.md`，`<slug>` 从文章标题或 URL 自动推导

---

## 工作流（4 个阶段）

### 阶段 0：初始化

- 接受输入：URL 使用 web_fetch 抓取内容；本地 `.md` 文件直接读取
- 从内容中提取文章标题
- 推导 `<slug>`：将标题转为小写连字符格式（如 `karpathy-llm-wiki`）
- 创建输出文件 `reading/<slug>.md`，写入文件头：

```markdown
# <文章标题>

> 原文：<URL 或文件路径>
> 整理日期：YYYY-MM-DD

---
```

### 阶段 1：摘要

- 自动执行，无需用户触发
- 生成结构化中文摘要，追加写入文件：

```markdown
## 摘要

### 一句话概括
...

### 核心论点
...

### 关键概念
- 概念 A
- 概念 B
- 概念 C
```

- 写入完成后，在对话中告知用户文件已更新，展示摘要内容
- 等待用户指令：
  - 「继续翻译」→ 进入阶段 2
  - 「跳过翻译，我想下钻 X」→ 跳至阶段 3
  - 「完成」→ 退出

### 阶段 2：翻译

- 用户确认后执行
- 对原文进行逐段翻译，格式为原文段落 + 译文段落交替呈现
- 追加写入文件：

```markdown
## 翻译

**原文**
> ...

**译文**
...

---

**原文**
> ...

**译文**
...
```

- 写入完成后告知用户，等待指令：
  - 「我想下钻 X、Y」→ 进入阶段 3
  - 「完成」→ 退出

### 阶段 3：概念下钻

- 用户主动指定想深入了解的概念（可以一次指定多个）
- 对每个概念生成深度展开内容：背景、原理、与其他概念的关系、示例（如适用）
- 追加写入文件，每个概念一个子节：

```markdown
## 深度笔记

### <概念名>

...

### <概念名>

...
```

- 每个概念写入后，询问是否还有其他概念要下钻
- 用户说「完成」或「不需要了」时退出

## 文件完整结构示例

```markdown
# LLM Wiki 模式

> 原文：https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
> 整理日期：2026-05-06

---

## 摘要

### 一句话概括
用 LLM 代替人工持续维护结构化个人 Wiki，让知识从「每次从零检索」变成「持续复利积累」。

### 核心论点
- RAG 每次查询从零推导，没有知识积累
- LLM Wiki 把知识「编译」一次后持续维护
- 人类负责策划资料和提问，LLM 负责所有维护工作

### 关键概念
- LLM Wiki
- RAG 对比
- Schema 层
- Ingest / Query / Lint 三操作

---

## 翻译

**原文**
> Most people's experience with LLMs and documents looks like RAG...

**译文**
大多数人使用 LLM 处理文档的方式是 RAG...

---

## 深度笔记

### RAG 与 LLM Wiki 的本质区别

...
```

## 阶段跳过规则

| 用户说 | Skill 行为 |
|--------|-----------|
| 「继续」/ 「好」/ 「可以」 | 进入下一阶段 |
| 「跳过翻译」 | 跳过阶段 2，直接询问要下钻哪些概念 |
| 「完成」/ 「不需要了」/ 「结束」 | 退出，告知文件路径 |
| 在任意阶段指定概念名 | 识别为下钻请求，进入阶段 3 |

---

## 约束

- 所有输出内容使用中文
- 摘要保持精炼，不超过原文 20% 篇幅
- 翻译以段落为单位，保持原文结构
- 概念下钻聚焦于用户指定的概念，不随意扩展
- Skill 只写入 `reading/` 目录，不修改其他文件

---

## 与仓库的关系

此 skill 与当前仓库绑定，是个人知识网络的组成部分。`reading/` 目录下的文件是知识积累的原始输入层，未来可与 wiki 层、索引等配合使用（参考 LLM Wiki 模式）。
